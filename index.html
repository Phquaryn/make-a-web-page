Variables and Strings in Python

Variables:

A Python variable is a way to use a names to keep track of stored values. You name or label a number or a "string" of characters.

Lets look at assigning a variable to a number. Why would you do that? When we create our code, we might understand why we are using the number 1.21. But, if we assign the number an expression to describe it like this: 1.21 = gigawattsBackToTheFuture , the next person to read our code will understand its meaning, too. We could further clarify it with a comment that is readable by humans but not read by the interpreter or compiler. In python it would be preceded by the pound sign.

As the name implies, this variable can change, or vary. So if, in the future, the technology improves you may be able to propell yourself at gigawattsBackToTheFuture = gigawattsBackToTheFuture - 0.5, telling is that we need merely 0.71 gigawats to power our future-Delorean.

But variables aren't just integers. A string is a sequence of characters contained in quotation marks. One use of a string could be to create reusable text on your webpage. greeting = "Hello" just pop print greeting wherever you want to repeat the text "Hello" on your webpage.

Q: What is the difference btwn 2+2 and "2"+"2"? A: 2+2 are numbers (integers), "2"+"2" are strings, and you cannot add them.

You can index string, or find the position of something in a string using "find". Ask Python to find the e in the word 'test' like this:'test'.find('e') and you might think it will return the number 2, because to us the 'e' is the second letter. In fact, the computer is counting from zero, so it will return a 1. If you ask Python to find the q in the word 'test' like this:'test'.find('q'), it will return a -1, to let you know that letter doesn't exist in that string.

You can also index strings using square brackets. If we have a database with date of birth formatted like this: 01012001, and we wanted to pull just the year, we could assign the dob a variable like this: dob = '01012001' and then print just the year using square brackets like this: print dob[4:] Putting the 4 in front of the colon tells it to start at position four and leaving the other side of the colon blank tells it to run to the end. If you wanted to print just the month you would use print dob[0:2]. If I wanted to make sure my selection will work even if the dob format changes, I could use dob[-4:]. So even if my date format in my database changes to 01-Jan-2001, I will still get the last four digits and still capture the year as my result.

Functions:

A function has inputs (like the cameras showing the car the surroundings in the video) and outputs (both the 3D model and the self-driving car not hitting walls, other cars, pedestrians). In python, we can write a fuction like: 'def say_hello(name):' In this example, 'name' is a parameter or an input. We would tell our function name is 'Bob' by typing say_hello('Bob')The output would be whatever our function returns.

def helloGurl(name):
string = 'Hello ' + name

print helloGurl('Mary')
We've written our function, but will it work? It has all the pieces we need, 'def' to tell python the function has started, first line ending with a colon, the next line is indented properly, and we're printing the result after adding our string to our function, right? Wrong! This won't work because our function never returns anything. If we ran this, we would get an unexpected result. In this case instead of the 'Hello Mary' we are hoping for, it would return 'None'.

Simply adding return string as the third line of code (properly indented) and our function is in business.

Procedures or functions can also be built-into Python. '.find' is a built-in procedure. This is useful because whenever we need to use .find to identify something in a string, we have the ability to do this. When we write our own functions, we give ourselves the ability to do something anywhere in the project without rewriting history each time.
